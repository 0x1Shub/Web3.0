1) What is meant by Token Launchpad ??
A Token Launchpad is a platform that helps creators create, distribute, and raise funds for a new cryptocurrency token without having to build all the infrastructure from scratch.
Kickstarter for crypto projects, where startups launch their tokens and investors can buy them early.

2) What Does a Launchpad Usually Provide?
1. Token Creation
Creates a token on a blockchain such as: Solana, Ethereum, BNB Chain
You define:
Name: SOCIAL
Symbol: SOC
Decimals: 9
Supply: 1,000,000

2. Token Sale
Investors buy tokens before public launch.
Example: 1 SOC = $0.01
An investor pays: 100 USDC and receives: 10,000 SOC

3. Liquidity Creation
After the sale, the launchpad may automatically create a liquidity pool on a DEX.
For example: Raydium, Uniswap
This allows users to trade the token immediately.

4. Vesting
To prevent dumping, tokens can be locked.
Example: Investor bought 100,000 tokens
Month 1 -> 25,000 unlocked
Month 2 -> 25,000 unlocked
Month 3 -> 25,000 unlocked
Month 4 -> 25,000 unlocked

3) Types of Launchpads ??
- IDO (Initial DEX Offering): Token launches directly on a DEX.
Example flow:
Project
   ↓
Launchpad
   ↓
DEX Liquidity Pool
   ↓
Public Trading

- IEO (Initial Exchange Offering): A centralized exchange manages the token sale.
Examples include launches on exchanges such as:
Binance
Bybit

- ICO (Initial Coin Offering): The older model.
Project sells tokens directly from its website.

4) What is Transactions Vs Instructions in Solana ??
Transaction = The Envelope
A transaction is the container that is sent to the Solana network.
It contains:
Signatures
Fee payer
One or more instructions
Transaction
│
├── Signature
├── Fee Payer
├── Instruction 1
├── Instruction 2
└── Instruction 3

A transaction in nothing but the bunch of instructions.


Instruction = The Actual Task
An instruction tells a Solana program what to do.
Examples:
Transfer SOL
Create token
Mint token
Burn token
Create account

Think of an instruction as a function call:
transfer(
  from,
  to,
  amount)

When you create this transfer request, you're creating an instruction.

7) What is mint ??
A Mint Account is an account that defines a token.
It stores information like: Token Name (via metadata), Decimals, Total Supply, Mint Authority, Freeze Authority
Think of it as: Token Blueprint or Token Definition
A Mint is a special Solana account that defines a token and controls its supply. It is the source from which new tokens can be created ("minted").

A legitimate launchpad should never ask the user for:

Private Key
Seed Phrase
Recovery Phrase

If a website asks for any of those, treat it as highly suspicious.

The correct approach is:

Website creates transaction
        ↓
Wallet signs transaction
        ↓
Website sends signed transaction

The private key never leaves the wallet, which is why users can safely create tokens, transfer funds, and interact with smart contracts through wallets like Phantom without exposing their secret keys to the website.

12) What Is MINT_SIZE?
Inside SPL Token: MINT_SIZE  equals: 82 bytes
The function basically does: How much SOL is required to permanently store 82 bytes of data?
RPC calculates the answer.

Why Is Buffer Used?
This part: const data = Buffer.alloc(67);
creates raw binary data.
Solana programs understand: Bytes
not: {
  decimals: 9
}

So SPL Token converts: {
  decimals: 9,
  authority: ...
}

into: 0109ABCDEF...
(binary format)
and sends it.

15) Why partialSign?
Creating an account requires:
New Account Signature
Think: Create Account

which is: keypair
So: transaction.partialSign(keypair);
adds: Mint Account Signature

Real World Analogy

16) Why wallet.signTransaction()?
Because the transaction needs TWO signatures.
Signature #1
transaction.partialSign(keypair)

Signs as: Mint Account
Signature #2 wallet.signTransaction(transaction)

Signs as: Payer Wallet

Remember: fromPubkey: payer.publicKey
The payer is spending SOL.
Therefore payer MUST sign.

Create Mint Transaction

│
├── Create Account
│      └── Requires Mint Signature
│
├── Pay Rent
│      └── Requires Wallet Signature
│
└── Initialize Mint

17) What is token program vs system program ?
System Program manages SOL and accounts.
Token Program manages SPL tokens.

1. User opens Token Launchpad.

2. User connects Phantom wallet.

3. useWallet() gets wallet information from WalletProvider.

4. User enters token details.

5. Frontend generates a new Mint Keypair.

6. Transaction is created.

7. System Program creates an empty account
   that will become the Mint Account.

8. Token Program initializes the Mint Account
   and stores decimals, authorities and supply information.

9. Mint Keypair partially signs because a new account
   is being created.

10. User Wallet signs because it is paying
    the rent exemption and transaction fees.

11. Signed transaction is sent to RPC.

12. Validators execute the transaction.

13. Mint Account is created on-chain.

14. Token now exists (Supply = 0).

15. Later mintTo() can be called to create
    actual token supply.
